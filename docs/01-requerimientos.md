# Requerimientos del sistema Geo Events

## Descripción general

Aplicación web orientada a servicios que permite registrar y consultar eventos urbanos geolocalizados.

## Actores

- Ciudadano
- Administrador (opcional)

## Requerimientos funcionales (ejemplos)

- RF1. Registrar evento con título, descripción, categoría y geolocalización.
- RF2. Consultar lista de eventos.
- RF3. Actualizar datos de un evento.
- RF4. Eliminar evento.

## Requerimientos no funcionales (ejemplos)

- RNF1. La API debe exponer endpoints REST y usar JSON.
- RNF2. Código organizado por capas (rutas, controladores, modelos).
- RNF3. Debe existir documentación técnica de la API.
